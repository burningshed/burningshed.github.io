First Steps

So first, after reading about the setup in this medium post.1 I wanted to set up org-mode to publish files in a similar way - that from a set of org-files generate a blog post.

The setup described has all of the blog post "source" files grouped into a folder, and then converted to html using org-publish. This keeps all of the pages of the page together, only updating new/changed ones. It does allow you to automatically upload them and run other commands during publishing.

org-customize

for each project, you need an entry in org-publish-project-alist - to create/add to this list, I used the command "org-customize" this brings up a list of org-mode option groups, enter expands the group, I found the setting I needed to change under org export -> org publishing. [insert image: org-customize]

org-customize usings the emacs easy-customization layer, and since this entry is an alist (associated list) you add entries to the list. For this setting, each entry is one project - the key is the project title, and the value is all of the settings, in the format (:setting value :setting value) etc, the () was essential to making it work. The org-mode manual states the publishing-function should be org-publish-to-html, this is incorrect as that function no longer exists in current releases. I believe the correct equivilant function is org-html-publish-to-html.

publishing a simple page

By the end, I found it easier to work with my .spacemacs file directly, particularly when debugging issues. This has the added advantage of resulting in a more readable file, as whitespace can be customized. I ended up with the following: 

'(org-publish-project-alist
  (quote
   (
    ("posts"
     :base-directory "~/projects/myWeb/blog/src/"
     :publishing-directory "~/projects/myWeb/blog/publish/"
     :publishing-function org-html-publish-to-html
     :headline-levels 4
     :recursive t
     :auto-preamble t
     )
   ("org-static"
     :base-directory "~/projects/myWeb/blog/src"
     :base-extension "css\\|js\\|png\\|jpg\\|gif\\|pdf\\|mp3\\|ogg\\|swf"
     :publishing-directory "~/projects/myWeb/blog/publish/"
     :recursive t
     :publishing-function org-publish-attachment
     )
   )))

this went in the (custom-set-variables …) section

This problem continued until I added a few more lines to the configuration, following the tutorial at worg.2 Their example was ("org-notes" :base-directory "~/org/" :base-extension "org" :publishing-directory "~/publichtml/" :recursive t :publishing-function org-html-publish-to-html :headline-levels 4 :auto-preamble t) to keep it consistant with the .spacemacs format I had to change it to (quote(("org-notes1" …) ("org notes2" …)))

With further testing, it still works even when I returned to my old function? The publishing function was my issue, I was using org-html-export-to-html not org-html-publish-to-html

Things I learned along the way

using footnotes in org-mode (here, used for citations)

can be done by number C-c C-x f is the quick command for them 3 in text format is [fn:name] placed in text and again at the bottom of the document

this can also be done entirely in-line with [fn:name:def]4

code blocks

inline code blocks can be done via src_<lang_id>[header]{code} multi-line code blocks can be done via the following: 

#+NAME: <id>
#+BEGIN_SRC <lang_id> <args>
 <body>
#+END_SRC
